## 社区维护

Capacitor 自诞生以来始终是 100% 开源（MIT 许可），并将一直保持开源。开发者可以通过 Capacitor 的社区维护策略，确信 Capacitor 是其跨平台应用的正确选择。Capacitor 团队定期发布新版本、修复错误，并非常欢迎社区提交拉取请求。

## Capacitor 维护与支持状态

考虑到时间和资源的现实限制，以及需要支持不断发展的移动平台生态，随着时间的推移，Capacitor 团队有必要发布新的主版本。然而，Capacitor 将尽一切努力使向新版本的过渡尽可能平稳。Capacitor 团队建议更新到最新版本的 Capacitor，以获取最新的功能、改进和稳定性更新。

各 Capacitor 版本的当前状态如下：

| 版本 |     状态     |     发布日期     | 维护结束时间  | 扩展支持结束时间 |
| :---: | :----------: | :--------------: | :-----------: | :--------------: |
|  v7   |    维护中    |  2025年1月20日   | 2026年6月8日  |  2026年12月8日   |
|  v6   |  扩展支持   |  2024年4月15日   | 2025年7月20日 |  2026年1月20日   |
|  v5   | 终止支持 |   2023年5月3日   | 2024年10月15日 |  2025年4月15日   |
|  v4   | 终止支持 |  2022年7月27日   | 2023年11月3日 |   2024年5月3日   |
|  v3   | 终止支持 |  2021年5月18日   | 2023年1月27日 |  2023年7月27日   |
|  v2   | 终止支持 |  2020年4月3日   | 2021年11月18日 |  2022年5月18日   |
|  v1   | 终止支持 |  2019年5月23日   | 2020年10月3日 |  2021年4月3日   |

- **维护中**：仅提供关键错误和安全修复。不进行主要功能改进。
- **扩展支持**：不再向新客户提供。

## 最低支持平台

每个 Capacitor 版本都会为其支持的平台设定最低版本要求。团队致力于支持尽可能广泛的平台版本，在每个主版本发布时，会根据使用情况和技术限制评估各平台的最低版本要求。
