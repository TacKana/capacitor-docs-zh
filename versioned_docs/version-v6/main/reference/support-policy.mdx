## 社区维护

Capacitor 自诞生伊始就采用 100% 开源（MIT 协议）模式，并将始终保持这一原则。通过 Capacitor 的社区维护策略，开发者可以确信 Capacitor 是其跨平台应用开发的正确选择。Capacitor 团队定期发布新版本和错误修复，并热忱欢迎社区提交 Pull Request。

## Capacitor 维护与支持状态

考虑到时间和资源有限的现实情况，以及支持不断发展的移动平台生态系统的需求，Capacitor 团队需要适时发布新的主版本。不过，Capacitor 会尽一切努力确保向新版本的过渡尽可能平稳。Capacitor 团队建议升级到最新版本，以获得最新的功能、改进和稳定性更新。

各 Capacitor 版本的当前状态如下：

| 版本 | 状态 | 发布日期 | 维护结束时间 | 扩展支持结束时间 |
| :---: | :---: | :---: | :---: | :---: |
| V6 | 支持终止 | 2024年4月15日 | 2025年7月20日 | 2026年1月20日 |
| V5 | 支持终止 | 2023年5月3日 | 2024年10月15日 | 2025年4月15日 |
| V4 | 支持终止 | 2022年7月27日 | 2023年11月3日 | 2024年5月3日 |
| V3 | 支持终止 | 2021年5月18日 | 2023年1月27日 | 2023年7月27日 |
| V2 | 支持终止 | 2020年4月3日 | 2021年11月18日 | 2022年5月18日 |
| V1 | 支持终止 | 2019年5月23日 | 2020年10月3日 | 2021年4月3日 |

- **维护期**：仅修复关键错误和安全问题，不提供重大功能改进。
- **扩展支持**：对于需要额外长期支持的团队和组织，Capacitor 提供扩展支持选项。了解更多信息，请查看我们的 [企业服务](https://capacitorjs.com/enterprise)。

## 最低支持平台

每个 Capacitor 版本都为其支持的平台设定了最低版本要求。Capacitor 团队致力于支持尽可能广泛的版本范围。在每个主版本发布时，团队会根据使用情况和技术限制评估各平台的最低版本，并据此进行必要的调整。
