## 社区维护

Capacitor 从诞生之初就是 100% 开源（MIT 协议），并且将一直保持开源。开发者可以通过 Capacitor 的社区维护策略，确保 Capacitor 是其跨平台应用的正确选择。Capacitor 团队定期发布新版本、修复错误，并且非常欢迎社区的拉取请求。

## Capacitor 维护与支持状态

考虑到时间和资源的现实限制，以及需要支持不断发展的移动平台生态，Capacitor 团队有必要随着时间的推移发布新的主版本。然而，Capacitor 会尽一切努力使向新版本的过渡尽可能平稳。Capacitor 团队建议更新到最新版本的 Capacitor，以获得最新的功能、改进和稳定性更新。

各 Capacitor 版本的当前状态如下：

| 版本 |      状态      |    发布日期    |   维护结束时间   |   延长支持结束时间  |
| :---: | :------------: | :------------: | :--------------: | :----------------: |
|  V5   |   支持已结束   |  2023年5月3日  | 2024年10月15日 |  2025年4月15日   |
|  V4   |   支持已结束   | 2022年7月27日 | 2023年11月3日  |  2024年5月3日    |
|  V3   |   支持已结束   | 2021年5月18日 | 2023年1月27日  |  2023年7月27日   |
|  V2   |   支持已结束   | 2020年4月3日  | 2021年11月18日 |  2022年5月18日   |
|  V1   |   支持已结束   | 2019年5月23日 | 2020年10月3日  |  2021年4月3日    |

- **维护期**：仅修复关键错误和安全问题。不进行重大功能改进。
- **延长支持期**：对于需要额外长期支持的团队和组织，Capacitor 提供延长支持选项。欲了解更多信息，请参阅我们的 [企业版服务](https://capacitorjs.com/enterprise)。

## 最低支持的平台版本

每个 Capacitor 版本为其支持的每个平台设定了最低版本要求。Capacitor 团队致力于支持尽可能广泛的版本范围。在每个主要版本发布时，团队会根据使用情况和技术限制，评估每个平台的最低版本并做出必要的调整。
