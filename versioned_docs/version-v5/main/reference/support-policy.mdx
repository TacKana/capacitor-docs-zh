## 社区维护

Capacitor 从一开始就是 100% 开源（MIT）的，并将永远如此。通过 Capacitor 的社区维护策略，开发者可以确保 Capacitor 是其跨平台应用的正确选择。Capacitor 团队定期发布新版本、错误修复，并且非常欢迎社区的拉取请求。

## Capacitor 维护和支持状态

考虑到时间和资源限制的现实，以及支持不断发展的移动平台的需要，Capacitor 团队有必要不时发布新的主要版本。但是，Capacitor 将尽一切努力使向新版本的过渡尽可能平滑。Capacitor 团队建议更新到最新版本的 Capacitor 以获得最新的功能、改进和稳定性更新。

每个 Capacitor 版本的当前状态如下：

| 版本 |      状态      |   发布日期   |  维护结束    |  扩展支持结束 |
| :-----: | :--------------: | :-----------: | :---------------: | :---------------: |
|   V5    |  支持已结束      |  2023年5月3日  |  2024年10月15日   |  2025年4月15日    |
|   V4    |  支持已结束      | 2022年7月27日  |  2023年11月3日    |  2024年5月3日     |
|   V3    |  支持已结束      | 2021年5月18日  |  2023年1月27日    |  2023年7月27日    |
|   V2    |  支持已结束      | 2020年4月3日   |  2021年11月18日   |  2022年5月18日    |
|   V1    |  支持已结束      | 2019年5月23日  |  2020年10月3日    |  2021年4月3日     |

- **维护**：仅进行关键错误和安全修复。没有主要功能改进。
- **扩展支持**：对于需要额外长期支持的团队和组织，Capacitor 提供扩展支持选项。要了解更多，请查看我们的[企业服务](https://capacitorjs.com/enterprise)。

## 兼容性建议

Capacitor 团队编制了一套关于将 Capacitor 与其他相关软件结合使用的建议。这并非全面列表，但涵盖了许多常见的兼容性问题。Capacitor 团队强烈建议每季度检查一次项目依赖项，以跟踪新版本、功能和错误修复。

## 最低支持平台

每个 Capacitor 版本为其支持的每个平台设置最低版本。Capacitor 团队努力支持尽可能广泛的版本范围。随着每个主要版本的发布，团队会评估每个平台的最低版本，并根据使用情况和技术限制进行必要的调整。
