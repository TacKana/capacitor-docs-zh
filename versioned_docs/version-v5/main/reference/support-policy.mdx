## 社区维护机制

Capacitor 自诞生起就是 100% 开源项目（MIT 协议），并将持续保持开源。通过 Capacitor 的社区维护策略，开发者可以确信 Capacitor 是其跨平台应用的正确选择。Capacitor 团队定期发布新版本、修复漏洞，并热忱欢迎社区提交代码贡献。

## 版本维护与支持状态

考虑到时间和资源的限制，以及需要支持不断演进的移动平台生态，Capacitor 团队会适时发布新的主版本。团队将竭尽全力确保版本升级过程平稳顺畅。我们建议开发者升级到最新版本，以获取最新功能、改进和稳定性更新。

各版本当前状态如下：

| 版本 |     支持状态     |   发布时间   | 维护终止日期  | 扩展支持终止日期 |
| :-----: | :--------------: | :-----------: | :---------------: | :---------------: |
|   V5    |  终止支持  |  2023年5月3日  | 2024年10月15日  |  2025年4月15日   |
|   V4    |  终止支持  | 2022年7月27日 | 2023年11月3日  |    2024年5月3日    |
|   V3    |  终止支持  | 2021年5月18日 | 2023年1月27日  |   2023年7月27日   |
|   V2    |  终止支持  | 2020年4月3日 | 2021年11月18日 |   2022年5月18日    |
|   V1    |  终止支持  | 2019年5月23日 |  2020年10月3日  |   2021年4月3日   |

- **维护期**：仅修复关键漏洞和安全问题，不提供重大功能改进
- **扩展支持**：对于需要长期支持的企业和组织，Capacitor 提供扩展支持选项。了解更多请参见[企业服务](https://capacitorjs.com/enterprise)

## 最低支持平台版本

每个 Capacitor 版本对其支持的平台设定了最低版本要求。团队致力于支持尽可能广泛的平台版本范围。每次主版本发布时，都会根据使用情况和技术限制重新评估各平台的最低版本要求。
