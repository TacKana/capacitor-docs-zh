将应用程序中的 JavaScript 和资源数量保持在最低限度，有助于提高启动和运行时的性能。

您可以使用扩展来分析哪些 JavaScript 包和资源导致了应用的**臃肿**，从而帮助决定是否应该使用或移除当前或新的依赖项。

## 包成本

点击 `配置` > `统计信息` 查看页面，其中会细分每个 JavaScript 包的大小。

- 您可以点击一个包来`展开`或`折叠`，以查看其中包含哪些子包。
- 点击某个包将跳转到编译后的 JavaScript 文件
- 包分析过程将以生产模式构建您的应用程序，并启用 source map。

:::note
该过程会显示所有构建的包。但这并不意味着所有包都会在运行时被加载。例如，`Ionic Core` 包显示了**所有** Ionic 组件，而您的应用在运行时只会加载它**使用**的组件。
:::

## 资源成本

点击 `配置` > `统计信息` 查看页面（您需要向下滚动），其中会细分应用中所有资源的大小。

这包括字体、图标、样式表、图片和其他文件。

使用此工具可以识别：
- 过大或包含您不需要的格式的字体（例如，当您已有 `woff2` 或 `woff` 版本时，还有 `ttf` 格式）
- 过大的图片，可能使用了不合适的图片格式（例如，对摄影图片使用了 `png`、`gif` 格式）

:::note
所有资源都会显示。但并非所有资源都会在运行时被加载。例如，[Ionicons](https://ionic.io/ionicons/) 可能显示数百个图标，但只有您在应用中使用的图标才会被加载。
:::
