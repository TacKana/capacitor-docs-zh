# 构建渐进式 Web 应用

Capacitor 对渐进式 Web 应用（PWA）提供一流的支持，使得构建一个能在 iOS 和 Android 上原生运行，同时也能在 Web 上作为移动 Web 应用或“渐进式 Web 应用”运行的应用变得轻而易举。

## 什么是渐进式 Web 应用？

简而言之，渐进式 Web 应用（PWA）是一种利用现代 Web 能力向用户提供类似应用体验的 Web 应用。这些应用部署在传统的 Web 服务器上，可以通过 URL 访问，并且可以被搜索引擎索引。

实际上，渐进式 Web 应用只是另一个术语，指的是为了移动端性能进行了优化，并利用新近可用的 Web API 来提供类似于传统原生应用功能（如推送通知和离线存储）的网站。

## Capacitor 与渐进式 Web 应用

Capacitor 对渐进式 Web 应用 _和_ 原生应用都提供一流的支持。这意味着 Capacitor 的桥接器支持在原生环境或 Web 环境中运行，许多插件在这两种环境中都可用，并且具有完全相同的 API 和调用约定。

这意味着您可以将 `@capacitor/core` 和 Capacitor 插件同时用作您的原生应用 _和_ 渐进式 Web 应用的依赖项，而 Capacitor 会在需要时无缝调用 Web 代码，在可用时调用原生代码。

此外，Capacitor 提供了许多实用工具来查询当前平台，以便在原生运行或在 Web 上运行时提供定制化的体验。

## 为您的应用添加渐进式 Web 应用支持

渐进式 Web 应用应具备一个应用清单（App Manifest）和一个服务工作者（Service Worker）。

### 应用清单

首先，您需要一个与应用清单文件，它位于您的 `index.html` 文件旁边，用于提供有关您应用的元数据，例如其名称、主题颜色和图标。例如，当您的应用被安装到主屏幕时，这些信息将被使用。

### 服务工作者

接下来，为了发送推送通知和离线存储数据，服务工作者将使您的 Web 应用能够代理网络请求并执行处理和同步数据所需的后台任务。

服务工作者功能强大，但也很复杂。通常不建议从头开始编写它们。相反，可以看看像 [Workbox](https://developers.google.com/web/tools/workbox/) 这样的工具，它们提供了常见的服务工作者方案，您可以轻松地将其整合到您的应用中。

有关使用服务工作者的更多信息，包括如何注册它们，请参阅 MDN 上的使用服务工作者页面。

## 渐进式 Web 应用性能

渐进式 Web 应用通过多个性能标准进行评估，包括交互时间和首次有效绘制。

在发布之前，请遵循渐进式 Web 应用检查清单，并使用 Lighthouse 来审计和测试您的应用。

如果您在使用现有前端技术栈时难以满足渐进式 Web 应用的性能标准，可以考虑将 Ionic Framework 作为一个选项，它能以近乎零配置的方式获得快速的 PWA 支持。
